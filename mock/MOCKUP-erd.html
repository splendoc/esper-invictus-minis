Soft Launch Scope: เฉพาะตารางที่จำเป็นสำหรับการใช้งานจริง — ไม่เพิ่มตารางที่ยังไม่ใช้

Access Rules (RLS)
Floor (staff): อ่าน + เขียน + แก้ไข patients และ visits ได้ทั้งหมด — ต้อง login
Public View: อ่านอย่างเดียว — เฉพาะ visits ที่ tab = 'waiting' หรือ 'active' + patient name (masked) — ไม่ต้อง login
No delete: ไม่ให้ลบข้อมูล — finalize only
Real-time Subscriptions
Floor: subscribe to visits table — เมื่อคอมพิวเตอร์เครื่องอื่นเปลี่ยนสถานะ → update ทันที
Public View: subscribe to visits table — เมื่อผู้ป่วยถูกลงทะเบียนหรือเปลี่ยนสถานะ → update ทันที
ทั้ง 5 เครื่อง เห็นข้อมูลเดียวกัน real-time

Wait / Stay Time Calculation
Wait time = now() - arrived_at (เมื่อ tab = 'waiting')
Stay time = now() - activated_at (เมื่อ tab = 'active')
ไม่ต้องเก็บ waitMin / stayMin ใน database — คำนวณ real-time จาก timestamp
ข้อดี: แม่นยำเสมอ ไม่ต้อง increment ทุกนาที
